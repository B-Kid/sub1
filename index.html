표준창

✅ 표준창 방식은 PAY-STORY 에서 제공한 창에서 결제방법(결제사)를 선택하고,

선택한 결제사의 인증화면을 통해 인증을 한 후 결제가 완료되는 일반적인 결제 방식 입니다.

 

목차

  1. 연동개요

  2. 결제연동

 

 

1. 연동개요

  • 서비스 구조와 결제 처리

    • 가맹점에서 SHA256을 이용하여 위변조 방지를 포함한 결제데이터를 생성합니다.

    • 결제페이지에서 form 데이터를 PAY-STORY API 로 HTTPS(SSL) Post 액션을 발생시켜 결제요청 데이터로 보냅니다.

    • PAY-STORY는 결제요청을 받아 사용자의 디바이스에 따른 결제창을 표시하고 사용자가 인증을 진행할 수 있도록 합니다.

    • 인증이 완료되면 PAY-STORY는 인증결과를 가맹점에 전달하고, 가맹점은 이를 이용하여 페이스토리의 결제 서버에 승인 요청을 합니다.

    • 승인이 완료되면, 리턴된 승인결과(notiUrl을 통해 전달받은 승인결과)를 가맹점의 결제 완료 페이지 표시합니다.

 

< OVER FLOW >

 

  • 암호화 구성

    • 인증/결제/취소 요청시 암호화 데이터로 요청 데이터의 무결성을 검증합니다.

 

 

2. 결제연동

  • 결제 요청 페이지 작성

    • merchantKey 값 설정

      • 결제 이용시 패스워드 기능을 하는 코드로 MID 번호별로 부여 됩니다.

      • 확인방법 : 가맹점 관리자 https://mms.minglepay.co.kr 에 접속하여 “가맹점정보 > 결제환경 설정 > KEY관리” 메뉴에서

        확인가능 합니다.

      • ❗ 가맹점 Key는 외부에 노출되지 않도록 주의합니다.

3. 취소 연동

  • 취소 요청 페이지 작성

    • 결제시 사용된 MID 와 merchantKey 를 세팅합니다.

    • 결제완료 이후 저장한 TID 를 PAY-STORY 서버로 전송하여 결제건을 취소합니다.

    • ✔ TimeOut 등으로 인해 TID 를 저장하지 못했다면, TID 대신 주문번호(ordNo)를 이용하여 취소가 가능합니다.

4. 결제 및 취소 데이터 Backend 수신

  • ✔ NotiUrl을 설정을 하시면, BackEnd 로 추가적인 승인 및 취소 결과 데이터를 수신 할 수 있습니다.

     

4.1 NOTIURL 설정

  • ✔ NotiUrl 설정은 가맹점 관리시스템(https://mms.minglepay.co.kr)의 가맹점정보=>결제환경 설정=>결제결과 통보관리 메뉴를 통해 설정 합니다.

  • ✔ 가맹점 관리시스템(https://mms.minglepay.co.kr)의 수신성공 기준 설정으로 가맹점 환경에 맞게 수신성공 처리를 하실 수 있습니다.

    • http 방식 : noti URL 에 접근시 http 코드가 200인 경우 수신 성공

    • json 방식 : noti URL 의 응답값이 json 형태의 {“resultCd”:”0000”} 인경우 수신 성공

  • ⚠ 가상계좌를 사용하시는 경우, NotiUrl 설정을 하셔야 입금 데이터 수신이 가능합니다.
